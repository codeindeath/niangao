# 年糕新版产品设计与现有实现差异清单 v4.1

日期：2026-05-24
原则：以新版产品设计为准，旧实现能用则用，不能用则删除，不因历史包袱造成新版实现混乱。

## 1. 总体结论

当前实现已经有可复用基础：

- React Native App 已有首页全屏经验卡片、卡片翻面、推荐/我的/收藏分页、记下页一级/二级领域点选、Apple 登录、聊聊基础对话、我的统计。
- Go 后端已有用户、经验、收藏、点赞、推荐、搜索、聊天、管理后台 API 基础。
- Admin Web 已有仪表盘、内容管理、用户管理、平台内容、审核队列、统计、系统配置和日志雏形。
- AI Service 已有人本主义风格 chat prompt 和 DeepSeek 调用雏形。

但当前实现的产品心智仍是旧版「经验分享 + 审核 + 点赞 + 首页/记录」结构。新版要变为「看看 / 聊聊 / 记下 / 我的」四主线，并引入完整的平台精选生产、私密数据后台管理、AI 功能治理和生产级数据模型。

因此需要做一次系统性重构，而不是在旧实现上补小功能。

## 2. 移动端差异

### 2.1 导航命名

现状：

- `home` 标签为「首页」。
- `chat` 标签为「对话」。
- `create` 标签为「记录」。
- `profile` 标签为「我的」。

新版：

- 看看。
- 聊聊。
- 记下。
- 我的。

处理：

- 改造。
- 底部导航保留四项结构，但全部改为新版命名。
- 图标也要从 emoji 改为统一图标体系或自定义轻图标，避免产品气质松散。

### 2.2 看看页

现状可复用：

- `HomeScreen.tsx` 已有推荐 / 我的 / 收藏分页。
- 已有全屏卡片。
- 已有上下滑切卡。
- 已有左右滑切分页。
- 已有点击翻面。
- 已有搜索入口。

主要差异：

- 分页顺序当前是推荐 / 我的 / 收藏，新版要求推荐 / 收藏 / 我的。
- 当前轻标签显示「来自年糕 / 用户原创」，新版要求「精选 / 原创」。
- 当前有点赞，旧字段为 `like_count`，新版是「有启发」。
- 当前卡片可能显示星级、score_reason、原文，前台新版不显示。
- 当前底部 actions 同时显示点赞和收藏，但命名和样式需改。
- 当前空状态有引导发布经验，新版推荐流不应过度引导记下。
- 当前「我的」和「收藏」没有列表工具态。

处理：

- 保留核心卡片交互和布局气质。
- 改造分页顺序、标签、反馈命名、卡片信息层级。
- 删除前台星级、score_reason、原文展示。
- 增加私密经验轻标签「仅自己可见」。
- 增加收藏/我的列表工具态。

### 2.3 ExperienceCard

现状可复用：

- 翻面动画。
- 全屏沉浸式卡片。
- 轻标签胶囊。
- 中央大字经验内容。
- 创建者区域。

主要差异：

- 正面不应显示「官」或「来自年糕」。
- 不应显示星级。
- 不应显示原文。
- 反面现在是纯文本 interpretation，新版需要结构化 items。
- 当前无话题独立呈现。
- 当前点赞按钮需要改为有启发。

处理：

- 保留卡片基础。
- 改造数据结构和渲染字段。
- 支持 `experience_interpretations.items`。
- 支持无解读状态。
- 支持聊天引用卡片进入同一详情卡片。

### 2.4 记下页

现状可复用：

- `CreateScreen.tsx` 中央输入区气质较好。
- 一级领域点选后展示二级领域。
- 字数统计。
- 公开 / 私密弱按钮。
- 话题编辑。

主要差异：

- 当前标题为「记录经验」，新版为「记下」。
- 当前保存按钮实际调用发布经验接口，反馈为「发布成功」，新版统一「已记下」。
- 当前内容限制 10-100 字，新版经验内容 100 字以内，但不应强制 10 字下限导致用户负担。
- 当前领域/子领域可选已符合新版。
- 当前没有「帮我整理」。
- 当前公开但审核失败会返回 review 信息，新版用户无感。

处理：

- 保留页面设计和领域交互。
- 改文案。
- 删除 10 字硬限制，保留空内容和 100 字上限。
- 增加「帮我整理」。
- 保存后统一反馈「已记下」。
- 后端自动处理公开失败转私密。

### 2.5 聊聊页

现状可复用：

- 基础聊天 UI。
- 人本主义 prompt。
- Go 后端编排聊天。
- 引用收藏经验上下文。

主要差异：

- 当前是单一 conversation，不支持议题。
- 当前 48h 历史窗口，不符合新版议题摘要 + 最近消息。
- 当前没有最近聊过。
- 当前没有议题标题生成、重命名、删除。
- 当前没有引用经验卡片 UI，只返回 referenced_experience_ids。
- 当前只用收藏经验，不完整支持用户自己的经验、收藏、公共高质量经验优先级。
- 当前标题「随便聊聊」、副标题「收藏的经验我都记着」不符合新版。

处理：

- 大改。
- 保留基础聊天气泡和 prompt 气质。
- 新增 chat_topics、chat_messages、chat_citations。
- 新增最近聊过和历史议题继续。
- 新增引用卡片组件。
- AI 上下文改为议题摘要 + 最近消息 + 相关经验。

### 2.6 我的页

现状可复用：

- 统计展示基础。
- 展示名/昵称区域。
- 领域分布视觉有部分参考价值。

主要差异：

- 当前「我发布的 / 我看过的 / 对话总数 / 经验包 / 对话人格 / 设置」是旧心智。
- 新版「我的」不是内容列表，而是个人主页与价值反馈。
- 需要展示经验资产、贡献反馈、自我变化。
- 第一阶段不做对话人格、经验包。
- 设置只保留意见反馈、退出登录、注销账号。
- 个人信息入口应在展示名附近，不放设置。

处理：

- 大改页面结构。
- 统计组件可复用一部分。
- 删除经验包、对话人格入口。
- 增加个人信息编辑入口。

### 2.7 搜索

现状：

- 已有搜索页面和搜索卡片页面。
- 当前搜索经验内容和创建者。

新版差异：

- 提示语改为「说说你的处境，或搜经验/创作者」。
- 搜索结果用列表。
- 点开后进入本次搜索结果集全屏卡片。
- 左右切页禁用。
- 需要支持处境搜索，后续可接 recommendation_ai / 向量。

处理：

- 改造。
- 现有搜索 API 可作为关键词搜索基础。

## 3. 后端差异

### 3.1 经验模型

现状：

- `experiences` 同时存 content、interpretation、domain、sub_domain、topics、is_private、review_status、quality_score、source_type、creator_name、original_text 等。
- `likes` 表用于点赞。
- `bookmarks` 表用于收藏。
- `review_status` 承载 pending/approved/rejected/private。

新版：

- `experiences` 统一承载精选、原创公开、原创私密。
- `visibility` 和 `lifecycle_status` 分开。
- 解读独立到 `experience_interpretations`。
- 点赞改为有启发，语义和表名都要调整。
- 收藏改为 `experience_collections`。
- 行为事件进入 `experience_events`。
- 聚合表现进入 `experience_metrics`。

处理：

- 大改数据库 schema。
- 旧字段可迁移，但不应继续混用 review_status 表达前台状态。
- `likes` 可迁移为 `experience_inspirations`。
- `bookmarks` 可迁移为 `experience_collections`。

### 3.2 发布/记下流程

现状：

- 公开经验同步执行 normalize、去重、hard_policy、AI review、翻译、解读。
- 审核不通过会给用户返回 rejected 和原因。

新版：

- 记下保存优先，反馈统一「已记下」。
- 公开但不适合公开时自动私密，用户无感。
- 用户主动「帮我整理」才整理。
- 私密经验不生成解读。

处理：

- 大改 API 语义。
- 把公开审核改成后台异步 pipeline。
- 删除向用户返回审核失败原因的前台逻辑。

### 3.3 推荐

现状：

- `GetRecommendations` 基于偏好、热度和收藏过滤。
- 主要使用 review_status approved + is_private false。

新版：

- 推荐需要 public + active + recommendation_status eligible。
- 使用 quality_tier、recommendation_score、行为事件、创作者打散、来源打散。
- 新用户早期偏平台精选。

处理：

- 改造。
- 现有推荐可作为 fallback。

### 3.4 聊聊

现状：

- 单 conversation。
- 48h messages。
- bookmark experiences 按 inferredDomain 取 50 条。
- AI prompt 用最近 40 条历史。

新版：

- 多议题。
- 临时会话。
- 议题摘要。
- 相关经验召回：自己经验、收藏、公共高质量。
- 引用卡片持久化。
- 清楚一点反馈。

处理：

- 大改。
- 当前 prompt 气质可保留并重写细节。
- `conversation` / `messages` 可迁移到 `chat_topics` / `chat_messages`。

### 3.5 AI 调用

现状：

- Go handler 直接调用 AI service。
- AI 统计较粗。
- 没有功能级 key alias、队列、限流、预算。

新版：

- AI Gateway。
- `ai_key_aliases`、`ai_function_configs`、`ai_call_logs`、`ai_jobs`、`ai_rate_limit_counters`、`ai_budget_policies`、`ai_usage_daily_stats`。
- 用户请求和内容生产分队列。

处理：

- 新增基础设施。
- 旧 AI 调用封装迁移到 Gateway。

## 4. 管理后台差异

### 4.1 信息架构

现状：

- Dashboard。
- ReviewQueue。
- ContentManagement。
- UserManagement。
- PlatformContent。
- DomainManagement。
- SystemConfig。
- AdminLogs。
- Statistics。

新版：

- 运营总览。
- 数据看板。
- 经验管理。
- 内容生产。
- 用户与反馈。
- AI 与系统。

处理：

- 大改导航。
- 可复用旧页面局部组件和 API client。
- 删除旧审核队列作为中心模块的心智。

### 4.2 内容管理

现状：

- 内容管理列表字段偏旧审核、点赞、收藏。
- 用户和平台内容分开页面。

新版：

- 经验管理统一承载精选、原创公开、原创私密。
- 平台精选和公开原创是公共内容运营对象。
- 原创私密进入后台但分区管理。
- 详情页要有前台预览、质量分层、数据表现、来源追溯、操作日志。

处理：

- 大改。
- `ContentManagement` 和 `PlatformContent` 可合并为新版经验管理 + 内容生产模块。

### 4.3 内容生产

现状：

- 平台内容支持手动新建、CSV 导入、批量 AI 评分。
- 没有素材库、批次、候选池、创作者池、来源池、周报、任务日志。

新版：

- 内容生产作为独立模块。
- 围绕素材库、生产批次、候选经验、创作者池、来源池、采集名单、内容周报、任务日志。

处理：

- 新增。
- 旧 PlatformContent 中新建和 CSV 导入可迁移。

### 4.4 用户与反馈

现状：

- 用户管理偏账号信息和统计。
- 私密经验可能在用户经验列表可见，但没有新版边界和审计。

新版：

- 用户详情可查看私密经验、私密议题、聊天记录、后台轻画像。
- 私密内容默认折叠，查看留审计。
- 反馈管理内置但不做工单系统。

处理：

- 大改。
- 新增审计。

### 4.5 AI 与系统

现状：

- 系统配置和 AI 状态较简单。

新版：

- AI 功能配置。
- key alias。
- 调用日志。
- 队列。
- 限流预算。
- 聚合看板。

处理：

- 新增。
- 旧系统配置只保留可用部分。

## 5. AI Service 差异

现状可复用：

- DeepSeek 调用封装。
- 人本主义 prompt 基础。
- chat prompt 风格接近新版方向。

主要差异：

- Prompt 里仍说“用户收藏的经验”，新版要包括自己经验、收藏经验和公共高质量经验。
- 当前 history 截取最近 40 条，不是议题摘要 + 最近消息。
- 当前引用逻辑不够产品化，没有卡片展示规则。
- 没有按 function_type 的 prompt version 管理。

处理：

- 保留语言风格基础。
- 重写 prompt 输入结构。
- 每个 AI 功能独立 prompt_version。
- AI service 不保存业务状态。

## 7. 删除或废弃清单

应删除或废弃：

- 前台「首页」命名。
- 前台「记录」命名。
- 前台点赞心智，改为「有启发」。

- 经验卡前台原文展示。
- 公开失败提示和原因展示。
- 前端 AI 生成解读按钮。

- 旧「对话人格」入口。

可保留但需改造：

- 翻面动画。
- 推荐/收藏/我的分页结构。
- 记下页视觉和领域/子领域点选。

- AI key alias 和限流统计。

- 聊聊议题体系。

## 8. 推荐实施顺序

1. 冻结新版文档。
2. 设计数据库迁移。
3. 重构经验模型和 API contract。
4. 改造移动端导航、看看、记下。
5. 重构聊聊议题、上下文和引用卡片。
6. 重构我的页。
7. 重构管理后台信息架构。
8. 接入 AI Gateway、队列、限流、统计。
9. 接入平台内容生产模块。
10. 全链路测试和火山引擎 staging 部署。
11. production 数据备份、迁移、发布。

## 9. 风险点

- 数据模型迁移范围大，需要先做 migration plan 和回滚计划。
- 聊聊从单 conversation 到多议题，前后端都要重构。
- 私密内容进入后台后必须有审计，否则后续风险不可控。
- 平台内容生产体系大，第一阶段要先实现最小闭环，不要一次做复杂采集平台。
- AI 成本如果不先拆 alias 和统计，后续无法判断真实成本结构。
- 旧文档和旧页面命名很多，实施时要明确废弃来源，避免产品语言混杂。
