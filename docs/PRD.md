## 1. 产品概述

### 1.1 产品定位

**年糕**（寓意"年年高"）是一个**结构化经验分享与 AI 个性化辅导社区**。

用户在年糕上：
- 创建和分享可行动的「经验」（非泛知识）
- 收藏、点赞有价值经验，社区投票筛选优质内容
- 基于自己的经验库，与 AI 进行人本主义风格的对话辅导

### 1.2 与竞品的核心差异

| 维度 | 抖音知识内容 | 知乎 | 小红书 | **年糕** |
|------|-------------|------|--------|---------|
| 内容形态 | 短视频 | 问答/文章 | 图文种草 | 结构化经验卡片 |
| 内容标准 | 完播率/点赞 | 专业度/赞同 | 种草效果 | **可行动性** |
| 用户行为 | 被动消费 | 获取答案 | 种草决策 | 积累+实践+成长 |
| AI 角色 | 推荐算法 | 无 | 无 | **基于用户经验的个性化辅导** |
| 内容沉淀 | 弱（刷走即忘） | 中 | 中 | 强（经验库持续复用） |

### 1.3 目标用户

**阶段一（冷启 - 前 6 个月）：**
- 职场 0-15 年，23-40 岁
- 对个人成长有主动需求
- 需要可执行的经验指导工作和生活
- 有记录和分享习惯的早期种子用户

**阶段二（增长 - 6-18 个月）：**
- 泛化至更广泛的自我提升人群
- 通过推荐系统和口碑传播获客

## 2. 核心功能

### 2.1 经验系统

#### 2.1.1 经验结构

每条经验包含两部分：

```
┌─────────────────────────────────┐
│  经验内容（必填，≤100 字）        │
│  "一句话说清楚要传达什么"         │
├─────────────────────────────────┤
│  经验解读（可选，≤500 字）        │
│  · 背景：这条经验从何而来         │
│  · 如何执行：具体步骤            │
│  · 适用场景：什么情况下有效       │
│  · AI 可辅助生成                │
└─────────────────────────────────┘
```

#### 2.1.2 经验操作

- **创建**：用户输入经验内容 → 可选填写/AI 生成解读 → 发布
- **浏览**：信息流（推荐/关注/热门）、搜索、分类浏览
- **互动**：点赞、收藏、分享
- **引用**：在解读中引用其他经验，形成经验网络

### 2.2 AI 辅导（核心差异化功能）

#### 2.2.1 对话基础

AI 底层采用**人本主义心理咨询**风格：
- 无条件积极关注
- 共情理解
- 不评判、不给命令式建议
- 引导用户自己发现答案

#### 2.2.2 经验锚定

AI 对话的关键设计：**对话始终锚定在用户的经验库上**。

- 用户发起对话时，AI 首先检索相关经验
- 对话中 AI 引用具体经验，如："你之前记录的「困难沟通先写提纲」这条经验，在这次和领导的谈话中能用上吗？"
- 用户收藏的他人经验同样纳入 AI 的知识库

#### 2.2.3 对话模式

| 模式 | 场景 | AI 行为 |
|------|------|---------|
| 困境分析 | 用户描述当前困境 | 帮助梳理问题，匹配相关经验 |
| 日常陪伴 | 用户随意聊聊 | 人本主义倾听，适时引入经验 |
| 经验复盘 | 用户想复盘某段经历 | 引导用户提炼可复用的经验 |
| 场景演练 | 用户想练习某场景 | 角色扮演，基于经验给出反馈 |

### 2.3 内容运营系统

#### 2.3.1 冷启内容

运营团队从以下来源清洗和结构化经验：
- 书籍（自我提升、管理学、心理学）
- 名人访谈/演讲
- 行业专业人士的公开文档
- 学术论文的实践结论

#### 2.3.2 内容审核

- AI 自动检测低质内容（纯搬运、广告、无操作性）
- 人工抽查
- 用户举报机制

### 2.4 推荐系统

基于用户行为的协同过滤 + 内容特征：
- 点赞、收藏、完整阅读、分享
- 经验完成率（多少人按经验去做了）
- 创建者信誉权重

## 3. MVP 范围

### 3.1 包含

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 用户注册/登录 | P0 | 手机号 + 验证码 |
| 经验创建 | P0 | 文本经验内容 + 解读 |
| 经验浏览 | P0 | 推荐流 + 搜索 |
| 点赞/收藏 | P0 | 基础互动 |
| AI 辅导对话 | P0 | 基于用户经验的 AI 对话 |
| 内容审核（AI 自动） | P0 | 发帖自动检测 |
| 运营后台 | P1 | 内容管理和审核 |
| 用户关注 | P1 | 关注关系 |
| 经验分类/标签 | P1 | 支持按领域浏览 |
| AI 生成经验解读 | P1 | 创建经验时的 AI 辅助 |
| 分享 | P2 | 生成分享卡片 |

### 3.2 不包含（留到后续版本）

- 视频/音频内容
- 直播
- 付费内容/打赏
- 社群/小组
- Web 端（先做移动端）

## 4. 成功指标

### 4.1 北极星指标

**每周被实际使用的经验数**（用户收藏后标记"已实践"的次数）

### 4.2 关键指标

| 指标 | 冷启阶段目标 | 增长阶段目标 |
|------|-------------|-------------|
| DAU | 1,000 | 50,000+ |
| 人均每日经验消费数 | 5 | 15 |

| 收藏/消费比 | 15% | 20% |
| AI 对话 DAU 渗透率 | 10% | 30% |
| 次日留存 | 30% | 45% |
| 人均 AI 对话轮次 | 3 | 8 |

## 5. 技术架构（概要）

### 5.1 整体架构

```
┌────────────────────────────────────────┐
│              移动端 App                 │
│         React Native / Flutter          │
└──────────────────┬─────────────────────┘
                   │
┌──────────────────▼─────────────────────┐
│           API Gateway (Kong)            │
└──────────────────┬─────────────────────┘
                   │
     ┌─────────────┼─────────────┐
     ▼             ▼             ▼
┌─────────┐ ┌──────────┐ ┌──────────┐
│ 用户服务 │ │ 经验服务  │ │  AI 服务  │
│ (Go)    │ │ (Go)     │ │ (Python) │
└────┬────┘ └────┬─────┘ └────┬─────┘
     │           │            │
     └───────────┼────────────┘
                 │
     ┌───────────▼────────────┐
     │   消息队列 (NATS/Kafka) │
     └───────────┬────────────┘
                 │
┌────────────────▼────────────────────────┐
│           数据层                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │PostgreSQL│ │  Redis   │ │Milvus/Qd │ │
│  │ (主数据)  │ │ (缓存)   │ │(向量检索) │ │
│  └──────────┘ └──────────┘ └──────────┘ │
└──────────────────────────────────────────┘
```

### 5.2 技术选型

| 层面 | 技术 | 理由 |
|------|------|------|
| 移动端 | React Native | 跨平台，生态成熟，团队招聘容易 |
| 后端 | Go (主服务) | 高并发，部署简单，性能稳定 |
| AI | Python (FastAPI) | LLM 生态集中在 Python |
| 数据库 | PostgreSQL | 结构化数据，成熟可靠 |
| 缓存 | Redis | Session、热点数据、限流 |
| 向量库 | Milvus / Qdrant | 经验语义检索 + RAG |
| 消息队列 | NATS | 轻量，适合微服务，比 Kafka 运维简单 |
| 对象存储 | MinIO / S3 | 图片（后续版本） |
| LLM | DeepSeek / Qwen API | 中文能力好，成本可控 |
