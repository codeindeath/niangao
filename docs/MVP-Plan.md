# 年糕 MVP 实施计划

> **版本:** v1.0 | **日期:** 2026-05-02 | **设计稿:** designs/Niangao-v3.html

**目标:** 实现年糕 App MVP，包含经验浏览/创建/互动、AI 人本主义对话辅导，可部署到生产环境支持 1000 DAU。

**架构:** React Native 移动端 → Go API 单体服务 → Supabase (PostgreSQL + Auth) → Python AI 服务 (DeepSeek API + RAG)

**总工期估算:** 12-14 周（单人开发，含测试和部署）

---

## 阶段 0：项目基础设施（第 1 周）

### 0.1 项目仓库和 CI/CD
- Git 仓库初始化，monorepo 结构
- GitHub Actions CI：lint、test、build
- 环境变量管理和密钥安全

### 0.2 Supabase 配置
- 创建 Supabase 项目
- 数据库 Schema 设计（用户、经验、收藏、点赞、对话记录）
- Row Level Security 策略
- 用户认证配置（手机号验证码登录）

### 0.3 开发环境
- Go 项目脚手架
- Python AI 服务脚手架（FastAPI）
- React Native 项目初始化

---

## 阶段 1：后端核心服务（第 2-4 周）

### 1.1 用户系统
- 手机号注册/登录（Supabase Auth 集成）
- JWT Token 管理
- 用户资料 CRUD

### 1.2 经验服务
- 经验创建（内容≤100字 + 解读≤500字）
- 经验 CRUD API
- 领域分类枚举
- 列表查询（分页、排序：最新/最热）

### 1.3 互动服务
- 点赞/取消点赞
- 收藏/取消收藏
- 点赞数和收藏数统计

### 1.4 数据库迁移和 Seed
- Schema 迁移脚本
- 冷启内容导入（从书籍/访谈清洗的种子经验）

---

## 阶段 2：AI 服务（第 4-6 周）

### 2.1 经验向量化
- 用户经验文本 → Embedding (DeepSeek API)
- 向量存储（Supabase pgvector）

### 2.2 对话引擎
- 对话 API（流式 SSE 响应）
- 人本主义 System Prompt 设计
- 经验检索：基于用户输入语义匹配用户经验库
- 对话中引用经验卡片

### 2.3 AI 辅助生成
- 经验解读自动生成（用户输入经验内容 → AI 生成解读）

---

## 阶段 3：移动端 App（第 6-11 周）

### 3.1 导航和基础架构
- 底部 Tab 导航（首页/对话/创建/搜索/我的）
- 网络层封装
- 状态管理

### 3.2 首页 — 经验信息流
- 经验卡片组件（作者头像、领域标签、内容、点赞收藏）
- 推荐列表 API 接入
- 下拉刷新、无限滚动
- 点赞/收藏交互

### 3.3 经验详情页
- 经验内容 + 解读展示
- 点赞/收藏/分享按钮

### 3.4 创建经验页
- 经验内容输入（100字限制 + 字数统计）
- 领域标签选择
- 经验解读输入（500字限制，可选）
- AI 辅助生成解读按钮
- 发布流程

### 3.5 AI 对话页
- 对话界面（聊天气泡，AI vs 用户）
- SSE 流式消息展示
- 经验引用卡片嵌入消息
- 输入框 + 发送

### 3.6 个人主页
- 用户信息展示
- 统计数据（经验数/收藏数/已实践数）
- 经验库列表
- 收藏列表
- 对话历史

### 3.7 搜索页
- 关键词搜索 API
- 搜索结果列表

---

## 阶段 4：内容审核和运营后台（第 10-11 周）

### 4.1 AI 自动审核
- 内容安全检测（敏感词、广告、低质）
- 发帖前自动审核

### 4.2 运营后台（轻量 Web）
- 内容列表 + 审核操作
- 冷启内容管理

---

## 阶段 5：测试、部署、上线（第 11-14 周）

### 5.1 测试
- 后端单元测试 + 集成测试
- API 压力测试（1000 并发）
- 移动端核心流程 E2E 测试

### 5.2 部署
- Go 服务部署到 Railway/Fly.io
- Python AI 服务部署
- 域名 + HTTPS 配置
- 监控和告警（基础）

### 5.3 应用商店上架
- App Store 审核准备
- Google Play 上架
- 软著申请（国内安卓渠道需要）
